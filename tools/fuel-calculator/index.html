Network Node Fuel Calculator

Paste node fuel snapshots, set an emergency floor and reserve window, and turn a limited fuel cache into an actionable dispatch brief. Critical nodes are stabilized to their target floor before the planner spends remaining fuel on the broader reserve target.

Fuel Snapshot

Format: name,currentFuel,maxFuel,burnPerHour[,deliveryDelayHours][,priority][,stabilityHours][,reserveHours]. Header rows with flexible column names/order, tab-separated exports, quoted CSV fields, formatted values like 1,200, extra spreadsheet columns, and # comments are accepted. If a row includes deliveryDelayHours, that node overrides the default delay for this run. If a row includes priority, higher values break ties inside the same alert band and critical-floor queue. If a row includes stabilityHours or reserveHours, that node carries its own coverage targets for this run. Trip capacity splits the dispatch order into sequential hauler manifests without changing alert-band precedence. Each node name must appear only once per run.
